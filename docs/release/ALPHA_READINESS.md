# GridFlux Alpha Readiness

## Alpha Scope

GridFlux is alpha-ready for a GridFlux-aware demo of GridFTP-like framed file
transfer:

- Framed STOR upload and RETR download through `gridflux-gridftp-server`.
- Bidirectional resume using `REST GFID:<transfer_id>` and manifest verified
  chunks.
- CRC32C chunk checksum by default, with `--checksum none` only for comparison.
- Control metadata commands: `SIZE`, `MDTM`, `CWD`, `CDUP`, `LIST`, and `NLST`.
- POSIX file IO default path, with optional experimental file-IO-only
  `io_uring`.
- Public export hygiene gate with private `AGENTS.md` excluded.
- Alpha artifact manifest and remote hash verification for release docs, gate
  JSON, private matrix CSV, and sidecar logs.

## Defaults

The alpha defaults are intentionally conservative:

- `file_io_backend=posix`
- `posix_write_strategy=auto`
- `file_io_buffer_size=0`
- `preallocate=off`
- `final_verify_policy=full`
- `manifest_flush_policy=every_n_chunks`
- `commit_sync_policy=none`

## Experimental Opt-ins

The following options remain explicit experiments and are not release defaults:

- `file_io_backend=io_uring`
- `preallocate=full`
- `final_verify_policy=verified_chunks`
- `manifest_flush_policy=final_only`
- `commit_sync_policy=fsync_file` or `fsync_file_and_dir`
- `posix_write_strategy=direct` or `coalesced`
- nonzero `file_io_buffer_size`

Phase 4L data did not justify a strong opt-in recommendation because spread
remains high across repeated 1GiB private runs.

## Not Beta Or Production Yet

GridFlux is not beta or production ready because:

- 1GiB private matrix throughput still has significant repeat-to-repeat spread.
- 100G dedicated-line validation is not complete.
- TLS/GSI/DCAU and production authentication are not implemented.
- Raw FTP STOR/RETR streams are not supported.
- Multi-file directory sync is not implemented.
- Long soak, packaging, and operational runbooks are not complete.

## Safety Notes

Public release must be made from an exported tree generated by
`tools/release/export_public_repo.py`. Do not publish local `AGENTS.md`, perf
results containing private topology, credentials, private keys, cookies, or
tokens.

Full alpha gate must also pass artifact sync verification. The generated
`tools/perf/results/<timestamp>_alpha-artifacts.json` is the source of truth for
required release artifacts; machine one and the remote validation machine must
match every required artifact hash before the gate is considered closed.
